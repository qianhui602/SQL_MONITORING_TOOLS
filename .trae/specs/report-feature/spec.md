# 报告功能规范

## 为什么做这个

当前系统已有丰富的数据采集和可视化能力，但缺乏能将这些数据整合成一份有分析价值、可导出的综合报告的能力。用户需要一键生成涵盖性能概况、死锁分析、慢查询、建议等的PDF报告，方便存档、分享和决策。

## 变更内容

- 新增报告数据聚合API：后端提供一个`/api/reports/summary`端点，按时间范围聚合所有模块数据
- 新增报告生成页面：前端新增`/report`路由，作为一个独立的报告生成与预览页面
- 新增DeepSeek AI报告分析：调用DeepSeek生成基于数据的分析结论和建议
- 新增PDF导出功能：前端使用`html2canvas` + `jspdf`将报告内容导出为PDF文件（图文并茂）
- 新增报告历史记录：保存已生成的报告记录，支持再次查看和下载

## 影响范围

- 影响的能力：数据报告生成、PDF导出、报告历史管理
- 影响的代码/系统：
  - 后端：新增`app/routers/reports.py`路由、新增`app/models/report.py`数据模型
  - 前端：新增`frontend/src/views/Report.vue`页面、新增`frontend/src/components/ReportChart.vue`组件
  - 依赖：前端新增`html2canvas` + `jspdf` npm包
  - 路由：前端新增`/report`路由，后端新增`/api/reports/*`端点

## 新增需求

### 需求 1：报告数据聚合API

系统后端SHALL提供一个报告数据聚合端点，按指定时间范围汇总各模块数据。

#### 场景 1.1：成功获取报告数据
- **WHEN** 用户选择时间范围（例如"最近24小时"）并请求生成报告
- **THEN** API返回聚合数据，包括：
  - 概览摘要：CPU/内存/连接数/缓存命中率等
  - 性能趋势：CPU/内存/连接数/IO趋势数据
  - 死锁统计：死锁次数、最新死锁时间、死锁详情（含AI分析）
  - 慢查询统计：慢查询数量、平均耗时、TOP慢查询
  - 阻塞统计：阻塞事件数
  - 磁盘统计：磁盘使用率
  - 索引分析：缺失索引数、碎片率高的索引数
  - AI分析结论：基于聚合数据自动生成的分析与建议

#### 场景 1.2：时间范围无效
- **WHEN** 用户选择的时间范围超过数据存储期限或无效
- **THEN** API返回400错误，提示时间范围无效

### 需求 2：报告生成页面

系统前端SHALL提供一个报告生成与预览页面，路径为`/report`。

#### 场景 2.1：页面加载
- **WHEN** 用户导航到报告页面
- **THEN** 页面显示：
  - 时间范围选择器（最近1小时/6小时/24小时/7天/自定义）
  - "生成报告"按钮
  - 报告内容区域（初始为空）

#### 场景 2.2：生成报告
- **WHEN** 用户选择时间范围并点击"生成报告"
- **THEN** 页面请求API获取数据，并在内容区域展示完整的报告内容，包括：
  - 报告标题和时间范围
  - 概览摘要卡片（CPU、内存、连接数、缓存命中率、死锁数、慢查询数）
  - 性能趋势图表（CPU、内存、连接数、IO）
  - 死锁分析：统计次数 + 表格列表
  - 慢查询分析：统计 + TOP列表
  - 阻塞/磁盘/索引状态
  - AI分析与建议区域

#### 场景 2.3：加载状态
- **WHEN** 报告数据正在加载中
- **THEN** 页面显示加载动画，禁用"生成报告"按钮
- **WHEN** 数据加载失败
- **THEN** 页面显示错误提示，允许重新生成

### 需求 3：PDF导出功能

系统SHALL支持将报告导出为PDF文件，导出的PDF包含图文内容。

#### 场景 3.1：导出PDF
- **WHEN** 用户点击"导出PDF"按钮
- **THEN** 系统执行以下步骤：
  1. 使用`html2canvas`将报告内容区域渲染为多张Canvas截图
  2. 使用`jspdf`将Canvas截图按页面大小分割并生成PDF
  3. 自动下载生成的PDF文件
- **THEN** 导出的PDF包含：
  - 所有概览摘要卡片
  - 所有趋势图表（图表以图片形式嵌入）
  - 死锁/慢查询表格数据
  - AI分析结论和建议
  - 页码和报告时间戳

#### 场景 3.2：导出过程中
- **WHEN** PDF正在生成中
- **THEN** 页面显示"正在导出PDF..."的加载提示
- **WHEN** 导出完成
- **THEN** 自动触发浏览器下载，提示恢复交互

### 需求 4：报告历史记录

系统SHALL支持保存和查看已生成的报告历史记录。

#### 场景 4.1：自动保存报告
- **WHEN** 用户成功生成一份报告
- **THEN** 系统自动将报告元数据保存到数据库（包含时间范围、生成时间、摘要数据等）

#### 场景 4.2：查看历史报告
- **WHEN** 用户在报告页面点击"历史记录"
- **THEN** 显示历史报告列表（生成时间、时间范围、概览摘要）
- **WHEN** 用户点击某条历史记录
- **THEN** 系统重新生成该报告的视图（从历史数据中恢复报告内容）

#### 场景 4.3：删除历史记录
- **WHEN** 用户删除一条历史记录
- **THEN** 该记录从数据库移除，列表更新

### 需求 5：DeepSeek AI分析建议

系统SHALL在报告中集成DeepSeek AI分析，提供专业的数据分析结论和优化建议。

#### 场景 5.1：AI分析成功
- **WHEN** 报告数据已聚合
- **THEN** 系统调用DeepSeek API，将聚合的性能数据作为上下文发送
- **THEN** AI返回：性能评估、潜在问题、优化建议（格式化为Markdown）
- **THEN** 报告结论区域显示AI分析结果

#### 场景 5.2：AI分析失败
- **WHEN** DeepSeek API调用失败或超时
- **THEN** 报告结论区域显示"AI分析暂时不可用"的提示，不影响报告其他内容
